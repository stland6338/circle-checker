# SPEC: circle-checker

> 作成日: 2026-03-28
> ステータス: Approved

---

## システム概要

### 一言で言うと

同人イベント「SHINY STAR FESTIVAL 10」のサークルリストを、スマホで快適に閲覧・ブックマークできる静的Webアプリ。

### 解決する課題

- 元サイトのサークルリストがテーブルレイアウトでスマホで見づらい
- 314サークルから気になるサークルを記録・管理する手段がない
- 当日の回る順を事前に計画できない

### 想定ユーザー

- 自分＋友人数人（URL共有で利用）
- ブックマークは各自のローカル端末に保存（共有不要）

## 機能一覧

| # | 機能名 | 説明 | 優先度 |
|---|--------|------|--------|
| 1 | サークル一覧表示 | カード型UIでスペース番号・サークル名・キャラ・外部リンクを表示 | P0 |
| 2 | ブックマーク | ★アイコンタップで追加/削除。localStorageに永続化 | P0 |
| 3 | キーワード検索 | サークル名・キャラクター名でリアルタイムフィルタ | P0 |
| 4 | 画面切替（下部ナビ） | 「全サークル」「ブックマーク」の2画面を下部タブで切り替え | P0 |
| 5 | キャラ別フィルター | キャラ名チップ/タブで絞り込み | P1 |
| 6 | 外部リンク遷移 | Pixiv・Xへのワンタップ遷移ボタン | P1 |
| 7 | ブックマーク並べ替え | ドラッグで順番変更（回る順の計画） | P2 |
| 8 | PWA対応 | オフラインキャッシュ、ホーム画面に追加 | P2 |

## 技術スタック

| レイヤー | 技術 | 選定理由 |
|----------|------|----------|
| フレームワーク | React 18 + Vite | 軽量・高速ビルド。314件のSPAに最適 |
| スタイリング | Tailwind CSS | モバイルファーストが容易。ユーティリティクラスで素早くUI構築 |
| データ取得 | Python + BeautifulSoup | 元サイトHTMLからサークル情報をスクレイピング→JSON化 |
| ホスティング | Vercel | 無料枠、GitHub連携デプロイ |
| PWA（P2） | vite-plugin-pwa | Service Worker自動生成でオフライン対応 |

---

## 制約・前提条件

- サーバーサイドなし（完全な静的SPA）
- データは事前にスクレイピングしてJSONファイルとしてバンドル
- ブックマークデータはlocalStorage（ユーザー間で共有しない）
- 今回のイベント専用（汎用化は考えない）
- イベント開催日: 2026/3/29（デプロイ期限）

## 画面・UIイメージ

### 画面構成

```
┌──────────────────────┐
│  🔍 検索バー          │
│  [キャラA][キャラB]... │  ← フィルターチップ（P1）
├──────────────────────┤
│ ┌──────────────────┐ │
│ │ サ|01  サークル名  ★│ │  ← カード（★でブックマーク）
│ │ キャラA, キャラB   │ │
│ │ [Pixiv] [X]       │ │
│ └──────────────────┘ │
│ ┌──────────────────┐ │
│ │ サ|02  サークル名  ☆│ │
│ │ ...               │ │
│ └──────────────────┘ │
│        ...            │
├──────────────────────┤
│  [全サークル] [★ブクマ] │  ← 下部タブナビ
└──────────────────────┘
```

### 操作フロー

1. アプリを開く → 全サークル一覧が表示される
2. 検索バーに入力 → リアルタイムで絞り込み
3. カードの★をタップ → ブックマーク追加（黄色に変化）
4. 下部タブで「ブックマーク」に切り替え → ブックマーク済み一覧
5. カードタップで詳細展開（Pixiv/Xリンク表示）

## データモデル（概要）

### Circle（サークル情報）— JSONファイルに格納

```typescript
interface Circle {
  id: number;              // 連番
  space: string;           // スペース番号（例: "サ|01"）
  name: string;            // サークル名
  characters: string[];    // 関連キャラクター名の配列
  section: string;         // セクション（ゲスト/サークル/企業等）
  links: {
    pixiv?: string;        // Pixiv URL
    twitter?: string;      // X(Twitter) URL
    other?: string;        // その他URL
  };
}
```

### Bookmark（ブックマーク）— localStorageに格納

```typescript
interface BookmarkData {
  ids: number[];           // ブックマーク済みサークルIDの配列
  order?: number[];        // 並べ替え順（P2）
}
```

## Q&A ログ

### Q1: アプリは今回専用？汎用？
**A:** 今回のイベント専用でOK。汎用化は考えない。

### Q2: ブックマークはユーザー間で共有する？
**A:** しない。各自のlocalStorageで完結。URLを共有すれば同じ一覧は見られる。

### Q3: UIパターンは？
**A:** カード型表示 + チェックリスト的なワンタップブックマーク + 下部ナビで一覧/ブックマーク切り替え。
